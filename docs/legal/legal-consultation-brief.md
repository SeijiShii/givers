# 弁護士相談のための概要メモ

**送付の目的**: 寄付プラットフォーム「GIVErS」の運営にあたり、想定している法的リスクと取っている（取りうる）対策を整理しました。ご相談内容を把握しやすくするため、**サービス概要・想定リスク・対応方針・実装済み対策・相談したい論点**を一つの文書にまとめています。

**最終更新**: 2025-02

---

## 1. このサービスがやりたいこと

### 1.1 サービス概要

- **GIVErS** は、**寄付による応援をつなぐウェブプラットフォーム**です。
- **作り手の GIVE**（見返りを求めず良いと思うものを作る）と**受け手の GIVE**（使って良かった人が応援したいと思う）をつなぐことを目指しています。
- **第一目標**: 日本で寄付の文化を根付かせること。日本での寄付に必要な情報（寄付金控除・領収書の扱い、日本語 FAQ 等）を最優先で提供します。

### 1.2 プラットフォームが「する」こと

| 項目 | 内容 |
|------|------|
| 決済の導線 | 寄付者は当プラットフォーム上のプロジェクトページから、Stripe 経由で寄付（単発・定期）を実行できる。 |
| 寄付金の流れ | **寄付金は Stripe Connect により、寄付者からプロジェクトオーナー（募集者）に直接入金**される。**プラットフォームは入金の経由地にならない**。 |
| 手数料 | **手数料を取らない**。寄付金からプラットフォームが手数料を徴収することはない（Stripe の決済手数料は別途発生）。 |
| プロジェクトの掲載・表示 | オーナーが登録したプロジェクト情報（説明・目標・コスト内訳等）を寄付者に表示する。 |
| ホスト権限 | 公式サイトを運営するホストは、ユーザー一覧の閲覧・**利用停止・停止解除**ができる。悪用と判断したアカウントの利用を停止し、当該ユーザーに紐づく**定期寄付（Stripe サブスク）を停止と同時に解約**する。 |
| 税額控除の「手助け」 | **税額控除の対象となる NPO 法人（認定NPO法人等）の場合のみ**、領収書等の発行を**手助けする**仕組み（「税額控除対象」の表明表示・寄付者氏名・住所の取得・オーナー向け寄付一覧のエクスポート等）を提供する。**書類の発行は行わない**（発行主体は受け取ったオーナー）。 |
| OSS・自ホスト | ソースコードを OSS として公開し、フォーク・自ホストを許容する。公式の URL と GitHub リンクを About 等に記載する。 |

### 1.3 プラットフォームが「しない」こと

- **寄付金の授受を仲立ちしない**（入金の経由地にならない）。
- **領収書・証明書等の書類を発行しない**（発行主体はオーナー）。
- プロジェクトの内容・オーナーの表明の**真実性・適法性を審査・保証しない**（方針として事前審査は行わず、事後対応として利用停止・凍結等を行う）。

## 2. 想定している法的リスク

運営者が**法的に不利になる可能性**や**詐欺幇助等の提訴を受けうる部分**として、以下の論点を想定しています。

### 2.1 公式サイト上での詐欺的プロジェクト（寄付の詐欺幇助）

- 寄付金はプラットフォームを経由せずオーナーに直接入金するが、**寄付の機会・決済導線を提供している**ため、虚偽のプロジェクトで寄付を集めた悪意あるオーナーがいた場合、「プラットフォームが詐欺を助長した」と指摘される余地がある。
- プロジェクトの**事前審査・本人確認（KYC）**は行わず、悪用と判断した場合の**利用停止は事後対応**。事前対策が弱いと「不十分な対策で被害を拡大した」と主張されるリスク。

### 2.2 「責任を負わない」公言の効力

- 悪用者が悪意を持ってクローンサービスを運用する場合、公式は責任を負わないと**公言**している。免責の明示はリスク低減に役立つが、**法律上どこまで有効かは法域・事案による**（消費者契約法、不法行為の故意・重過失による免責の制限等）。
- 公式サイト上で発生する詐欺・紛争と、クローン運営の免責は別論として、**公式運営についての免責・責任制限**も整理が必要。

### 2.3 税額控除・領収書まわり（誤表示・誤認助長）

- 「税額控除対象」の表明はオーナーが行い、プラットフォームは**無審査で表示**する想定。認定NPO等でないオーナーが「税額控除対象」と偽って表明した場合、寄付者が誤認し、**プラットフォームの表示・仕様がその誤認を助長した**と指摘される可能性。

### 2.4 利用停止・凍結の判断（恣意性と不作為）

- ホストが「悪用と判断した場合」に利用停止できるが、判断基準が**曖昧**だと、(1) **不当な利用停止**として訴えられる、(2) 逆に**明らかな悪用を見過ごした**として「不十分な対策」で訴えられる、の両方が論点になりうる。

### 2.5 本人確認・マネロン対策

- 認証は OAuth（Google/GitHub/Apple）・Email 等。**実名確認・本人確認（KYC）**の記載はない。寄付の受領者が法人・団体の場合、マネロン規制・犯罪収益移転防止法、資金決済法上の位置づけ等、必要に応じて専門家の確認が望ましい。

## 3. どのような対応を考えているか（方針・検討中の措置）

| リスク領域 | 対応方針・検討内容 |
|------------|-------------------|
| **詐欺幇助・仲介の範囲** | 利用規約で「プロジェクト内容・オーナーの表明についてプラットフォームは保証しない」「寄付の判断は寄付者自身の責任」等を明記。プロジェクト掲載時のガイドライン・禁止事項を定め、違反時の凍結・削除と併せて運用方針として残す。 |
| **免責・責任制限** | About・利用規約・README に「悪意あるクローン運営について公式は責任を負わない」旨および、プロジェクト・オーナー表明の保証をしない旨を記載。**文言は弁護士確認を想定**。 |
| **税額控除まわり** | 「税額控除の可否は寄付者・オーナーが各自で確認すること」「プラットフォームは表明の真正性を保証しない」旨を表示・利用規約に明記。認定番号等の簡易照合を行う場合は、範囲と責任の切り分けを専門家に確認。 |
| **利用停止・凍結** | 利用規約に「悪用」の定義・禁止行為の例と、停止・凍結の手続きを簡潔に記載。ホストの裁量を残しつつ、「合理的な根拠に基づく対応」が説明できるようにする。 |
| **第三者情報開示** | 裁判所の命令・法執行機関の照会等に備え、**開示の要否・範囲は弁護士等の助言に従う**前提で、管理画面から開示用データを出力する機能を用意（実装済み）。 |

## 4. すでに実装している対策

以下は、リスク低減または説明責任の一環として**すでに仕様・UI に反映している**ものです。

| 対策 | 内容 |
|------|------|
| **寄付金を仲立ちしないことの明示** | 寄付金は Stripe Connect でオーナーに直接入金することを、FAQ・About・仕様書で明示。資金の経由地にならないことを明確化。 |
| **利用停止時のサブスク解約** | 利用停止したユーザーに紐づく定期寄付（Stripe サブスク）は、停止と同時に解約する方針を文書化・実装。悪用者から継続入金させない。 |
| **利用停止・凍結時のメッセージ** | 利用停止アカウントで寄付・プロジェクト作成等を試みた場合、画面に「このアカウントは利用停止されています。ご不明な点は運営までお問い合わせください。」等を表示。凍結・削除プロジェクトに寄付等を試みた場合も「寄付の受付を停止しています。」「プロジェクトは終了しています。」等のメッセージを表示。単に 403 を返すだけでなく理由が分かるようにしている。 |
| **クローン・悪用について「責任を負わない」公言** | About ページに「悪用者が悪意を持って本ソフトウェアをクローンし、第三者により運営されるサービスについて、公式は責任を負わない」旨、および「開示の要否・範囲は弁護士等の助言に従う」旨を記載済み。 |
| **公式 URL・GitHub の明示** | About ページに公式サイトの URL と GitHub リポジトリへのリンクを記載。利用者が「公式かどうか」を参照できるようにしている。 |
| **開示用データの出力機能** | 第三者情報開示請求等に備え、管理画面から**ユーザーID または プロジェクトID を指定して開示用データを JSON で出力**する機能を実装。出力内容はユーザー情報・プロジェクト情報・寄付履歴等の定義済み範囲。開示の要否・範囲は弁護士等の助言に従う前提。 |

## 5. 弁護士に相談したい論点・確認したいこと

以下の点について、**法的な観点からの助言・文言の確認**をいただきたいと考えています。

### 5.1 利用規約・免責

1. **利用規約の整備**  
   仲介の範囲（寄付金を仲立ちしないこと）、プロジェクト・オーナー表明の保証をしない旨、免責・責任制限、禁止行為・利用停止の根拠を記載した利用規約の**文案・文言**について、問題がないか、不足している論点がないかの確認。

2. **免責条項の有効性**  
   「寄付の判断は寄付者自身の責任」「プロジェクトの内容・真実性を保証しない」等の免責・責任制限が、**消費者契約法・不法行為等の観点からどこまで有効か**。故意・重過失による免責の制限等の留意点。

3. **クローン・悪用についての免責**  
   About・README に記載する「悪意あるクローン運営について公式は責任を負わない」旨の**文言**が、注意義務論等で考慮されうるか、修正・追加の推奨があれば教えていただきたい。

### 5.2 詐欺幇助・仲介の範囲

4. **プラットフォームの位置づけ**  
   寄付金を経由せず決済導線のみ提供する場合、**詐欺幇助・仲介責任の観点**でどのような論点がありうるか。事前審査を行わず事後対応（利用停止）に頼る方針について、取るべき記載・運用上の注意点。

### 5.3 税額控除・表明

5. **税額控除まわりの表示**  
   「税額控除の可否は寄付者・オーナーが各自で確認すること」「プラットフォームは表明の真正性を保証しない」旨を表示・規約に明記する場合、**不足している論点や推奨される文言**の有無。

### 5.4 利用停止・手続き

6. **「悪用」の定義・利用停止の手続き**  
   利用規約に禁止行為の例・利用停止の手続きを記載する場合、**恣意性や不作為を争われないために必要な記載**の程度（簡潔に済むか、もう一段の具体化が必要か）。

### 5.5 その他

7. **本人確認・マネロン**  
   現状、寄付受領者（オーナー）の本人確認・KYC は行わない方針。**サービス規模・法域に応じて、早い段階で検討すべき法的要件**があれば教えていただきたい。

8. **第三者情報開示**  
   裁判所の命令・法執行機関の照会等に基づく開示の要否・範囲について、**事前に用意しておくべきデータの範囲や、開示に際しての留意点**（例：弁護士の関与のタイミング、記録の残し方）があればご助言いただきたい。
