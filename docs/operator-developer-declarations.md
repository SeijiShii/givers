# 運営者・開発者としての宣言（明記する内容）

弁護士相談の一歩手前として、**運営者および開発者として宣言・明記できる内容**をすべて列挙する。利用規約・About・README 等の文案化や、弁護士確認のたたき台とする。

**最終更新**: 2025-02

---

## 1. プラットフォームが「する」こと

- GIVErS は、寄付による応援をつなぐ**ウェブプラットフォームのソフトウェアを提供・運営**する。
- **決済の導線**を提供する。寄付者は当プラットフォーム上のプロジェクトページから、Stripe 経由で寄付（単発・定期）を実行できる。
- 寄付金は**Stripe Connect により、寄付者からプロジェクトオーナー（募集者）に直接入金**される。プラットフォームは決済処理の連携を提供するが、**入金の経由地にはならない**。
- **手数料を取らない**。寄付金からプラットフォームが手数料を徴収することはない（Stripe の決済手数料は別途発生する）。
- **プロジェクトの掲載・表示**を行う。オーナーが登録したプロジェクト情報（説明・目標・コスト内訳等）を寄付者に表示する。
- **ホスト権限**を運用する。公式サイトを運営するホストは、ユーザー一覧の閲覧・**利用停止・停止解除**ができる。悪用と判断したアカウントの利用を停止し、当該ユーザーに紐づく**定期寄付（Stripe サブスク）を停止と同時に解約**する。
- **利用停止されたアカウント**で寄付・プロジェクト作成等を試みた利用者には、画面で「このアカウントは利用停止されています。ご不明な点は運営までお問い合わせください。」等のメッセージを表示する。
- **凍結・削除されたプロジェクト**に寄付等を試みた利用者には、「このプロジェクトは現在、寄付の受付を停止しています。」「このプロジェクトは終了しています。」等のメッセージを表示する。
- **寄付行動の履歴**を、アカウントの有無を問わず記録する。匿名（トークン）寄付も記録し、アカウント作成後に引き継ぎを案内する。
- **税額控除の対象となる NPO 法人（認定NPO法人等）の場合のみ**、領収書等の発行を**手助けする**仕組み（「税額控除対象」の表明表示・寄付者氏名・住所の取得・オーナー向け寄付一覧のエクスポート等）を提供する。提供するのは手助けであり、書類の発行は行わない。
- **公式の URL と GitHub リポジトリへのリンク**を About ページ等に載せる。利用者が「公式かどうか」を参照できるようにする。
- ソースコードを **OSS として公開**し、**フォーク・自ホスト**を許容する。自ホスト版での「公式」表記の削除・変更は利用者に委ねる。

## 2. プラットフォームが「しない」こと・「経由地にならない」こと

- **寄付金の授受を仲立ちしない。** 寄付金は寄付者から Stripe Connect を通じてプロジェクトオーナーに直接入金され、**プラットフォームは入金の経由地とならない**。
- **領収書・証明書等の書類を発行しない。** 領収書等の発行主体は、寄付金を受け取ったオーナー（団体）である。プラットフォームは書類を発行しない。
- **手数料を取らない。** 寄付金からプラットフォームが手数料を徴収することはない。
- 認証バッジ・「公式サイトである」ことを証明するマークの発行は行わない（利用者が参照できる情報の提供にとどめる）。
- 偽サイトのリストの維持・通報窓口の提供・複雑な検証フローは、現状の運営方針として行わない。

---

## 3. プラットフォームが「保証しない」こと・「責任を負わない」と明記する内容

- **プロジェクトの内容・真実性・適法性を保証しない。** プロジェクトの説明・目標・費用等はオーナーが登録したものであり、プラットフォームはその正確性・真正性を審査・保証しない。
- **オーナー（募集者）の表明を保証しない。** オーナーが「認定NPO法人等である」「税額控除の対象である」等と表明した場合、その表明の真正性をプラットフォームは審査・保証しない。税額控除の可否は**寄付者およびオーナーが各自で確認する**ものとする。
- **寄付の判断は寄付者自身の責任**とする。どのプロジェクトにいくら寄付するかは寄付者が判断し、その結果についてプラットフォームは責任を負わない。
- **悪用者が悪意を持って本ソフトウェアをクローンし、第三者により運営されるサービス**について、**公式（GIVErS の運営）は責任を負わない**。OSS として配布・フォークを許容する一方、第三者による悪用については関知しない旨を明示する。
- 自ホスト版・フォーク版で公式の URL やリンクを削除した場合も、その運営について公式は責任を負わない。

## 4. 利用停止・凍結・削除の方針（運営者として宣言する内容）

- **ホスト**は、プロジェクトオーナーがシステムを**悪用していると判断した場合**、当該アカウントを**利用停止**できる。停止解除もできる。
- 利用停止したユーザーに紐づく**定期寄付（Stripe サブスク）は、停止と同時に解約**する。悪用者から継続入金させないためである。
- **プロジェクトオーナー**は、自らのプロジェクトを**凍結**または**削除**（論理削除）できる。
  - **凍結時**: 新規寄付・新規サブスクの受付を停止する。既存サブスクは継続する。
  - **削除時**: 新規受付を停止し、既存サブスクも解約する。
- 利用停止・凍結・削除の判断は、ホストまたはオーナーの裁量に基づく。プラットフォームは、合理的な根拠に基づく対応として運用する。

---

## 5. データ・プライバシー（運営者として宣言する内容）

- **寄付履歴は永続保存を原則**とする。一定期間後の自動匿名化・自動削除は行わない。
- **アカウント削除依頼時**は、当該ユーザーに紐づく寄付レコードの寄付者情報を**匿名化**し、レコードは残す（集計・分析の整合性のため行の削除は行わない）。
- **利用停止時**は寄付履歴の内容は変更しない。停止解除時にそのまま利用を再開できる。
- **データ分析**は、個人を特定しない匿名集計およびプロジェクトオーナー向けの自プロジェクト集計までとする。個人を特定する情報の分析・第三者提供は行わない。
- 削除依頼には応じる方針を明示し、GDPR 等の削除権を考慮する。
- **第三者情報開示請求等**（裁判所の命令・法執行機関の照会・訴訟等）に備え、**管理画面から開示用データを出力する機能**を用意する。ユーザーID または プロジェクトID を指定し、定義した範囲の情報を JSON 等で出力する。開示の要否・範囲は弁護士等の助言に従う。詳細は `docs/legal-risk-considerations.md` 4・`docs/user-management-mock-plan.md` 1.4 を参照。

## 6. 税額控除・領収書まわり（運営者として宣言する内容）

- プラットフォームは**書類を発行しない**。領収書等は**受け取ったオーナー（団体）が発行**する。
- **税額控除の対象となる NPO 法人（認定NPO法人等）の場合のみ**、オーナーが領収書等を発行する際の**手助け**（「税額控除対象」の表明表示・寄付者氏名・住所の取得・オーナー向け寄付一覧のエクスポート等）を提供する。
- 「税額控除対象」の表明は**オーナーが行う**。プラットフォームはその表明の**真正性を審査・保証しない**。税額控除の可否は寄付者・オーナーが各自で確認する。
- 寄付をいったんプラットフォームが受け取り助成する形は取らない（入金は Stripe Connect でオーナーに直接）。

## 7. クローン・自ホスト・公式について（運営者として宣言する内容）

- **自ホストには寛大**であり、公式かどうかにはこだわらない。フォークして公式 URL やリンクを削除するならそれもあり、とする。
- **公式 GIVErS** の URL と GitHub リンクは About 等に記載する。利用者が参照できる程度にとどめる。
- **悪用者が悪意を持ってクローンサービスを運用する場合、公式（GIVErS の運営）は責任を負わない**ことを公言する。OSS として配布・フォークを許容する一方、第三者による悪用については関知しない旨を明示する。
- 上記は、利用規約・About・README に記載することを検討する。文言は弁護士確認を推奨する（`docs/legal-risk-considerations.md` 参照）。

## 8. 寄付者・オーナーへの期待（利用者に求めることとして明記する内容）

- **寄付者**: 寄付の相手・金額・税額控除の可否等を**各自で確認したうえで**寄付すること。寄付の判断は寄付者自身の責任とする。
- **オーナー（募集者)**: プロジェクトの内容・費用・税額控除の対象であることの表明等について、**真実かつ適法**に登録・表明すること。表明の真正性は各自の責任とする。
- 禁止行為（虚偽の表明・詐欺・法令違反・他者への侵害等）に該当する利用は認めない。該当する場合、利用停止・凍結・削除の対象とする。

---

## 9. 運営の透明性・存続（運営者として宣言する内容）

- GIVErS 自体が GIVE で運営される。まわらなくなったら世に受け入れられなかったということとし、原理を曲げてまで存続しない。
- ある日突然クローズする可能性もあるため、利用者に申し訳ない思いを残さないよう**透明性を保つ**。健全性（青・黄・赤の信号等）をホストページで表示する方針とする。
